# 角色操作手册（Role Playbook）

## AUTHOR

目标：完成投稿与版本提交。

操作步骤：

1. 登录
2. 进入 Author Manuscripts 页面
3. 新建稿件（标题、摘要等）
4. 上传 PDF 到目标版本
5. 点击提交

预期结果：

- 稿件进入待筛选/评审流程
- Author 可在列表中查看状态

## REVIEWER

目标：完成接稿与评审提交。

操作步骤：

1. 登录
2. 打开 Reviewer Assignments
3. 接受 assignment
4. 进入 Review Editor
5. 提交评审意见

预期结果：

- assignment 状态流转
- 评审报告可被 Chair 端聚合查询

## CHAIR

目标：管理流程与最终决策。

操作步骤：

1. 登录
2. 在 Screening Queue 发起 round
3. 在 round 下分配 reviewer
4. 查看冲突检查与 Agent 分析
5. 在 Decision Workbench 执行终审

预期结果：

- 稿件状态按流程推进
- 决策记录写入并可追溯

## ADMIN

目标：监控系统运行与 Agent 任务。

操作步骤：

1. 登录
2. 打开 Agent Monitor
3. 查看任务状态、失败项和结果情况

预期结果：

- 发现异常任务并反馈给开发/运维

## 演示常见问题

1. 无法登录：检查 demo seed 是否执行
2. 页面空白：检查 token 是否过期并重新登录
3. Agent 无结果：检查 API key 与 Agent 服务是否启动
